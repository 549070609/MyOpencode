## 提示词类别说明

### 1. 主系统提示词
定义 OpenCode 的核心身份、行为规范和工作流程。不同模型使用不同的主提示词以获得最佳效果。

### 2. 模式切换提示词
控制 AI 的操作权限：
- **计划模式**：只读，仅分析和规划
- **构建模式**：完整权限，可执行所有操作

### 3. 代理提示词
Oh-My-OpenCode 插件的专业代理系统：
- **西西弗斯**：主协调器，管理任务和其他代理
- **探索代理**：快速搜索代码库
- **图书馆员**：研究外部库和文档
- **神谕**：高级架构决策
- **文档编写者**：创建技术文档
- **前端工程师**：视觉和UI设计
- **多模态查看**：处理非文本文件

### 4. 工具提示词
定义各种工具的使用方式和最佳实践。

### 5. 钩子提示词
生命周期钩子注入的提示词：
- **Ultrawork 模式**：极限工作模式，最大化代理利用
- **待办继续**：自动继续未完成的任务
- **压缩注入**：上下文压缩时保留关键信息
- **关键词检测**：根据用户输入触发特定模式

### 6. 内置命令模板
复杂工作流程的完整定义：
- **/refactor**：6阶段智能重构
- **/init-deep**：分层 AGENTS.md 生成

---

## 使用建议

1. **了解模式**：根据任务需要选择计划模式或构建模式
2. **善用代理**：复杂任务应委派给专业代理
3. **遵循约定**：始终遵循项目的代码规范
4. **验证输出**：重要操作后运行诊断和测试
